# 短信服务对接说明

本文说明当前项目中**登录短信**与**忘记密码短信**的实现位置、调用链，以及如何将「演示环境仅打日志」替换为**真实短信平台**（阿里云、腾讯云等）。

**存储说明（已实现）**：验证码与 60 秒发送间隔已写入 **Redis**（`spring-boot-starter-data-redis`），键前缀 `ec:sms:`。本地需启动 Redis（默认 `127.0.0.1:6379`，可通过 `REDIS_HOST` / `REDIS_PORT` / `REDIS_PASSWORD` 覆盖）。

---

## 一、当前实现

### 1.1 接口与顺序

| 场景 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 登录 | POST | `/api/user/sms/send-login` | Body：`SendLoginSmsDTO`（`phone`、`captchaId`、`captchaCode`） |
| 登录 | POST | `/api/user/login/sms` | Body：`SmsLoginDTO` |
| 忘记密码 | POST | `/api/user/sms/send-forgot` | Body：`SendForgotSmsDTO`（`username`、`phone`、图形码） |
| 忘记密码 | POST | `/api/user/forgot` | Body：`ForgotPasswordDTO`（含短信码） |

JWT 已对 `captcha`、`sms/send-login`、`sms/send-forgot` 等路径放行，见 `WebMvcConfig`。

### 1.2 核心类

| 类 | 作用 |
|----|------|
| `com.ecommerce.service.SmsCodeService` | 登录 / 忘记密码 发短信与校验 |
| `com.ecommerce.service.impl.RedisSmsCodeServiceImpl` | **Redis 实现**：`ec:sms:login:code:{phone}`、`ec:sms:forgot:code:{phone}`，TTL 5 分钟；冷却键 `ec:sms:login:cool:{phone}` / `ec:sms:forgot:cool:{phone}`，60 秒 |
| `com.ecommerce.service.impl.UserServiceImpl` | `sendLoginSms`、`sendForgotSms`、`loginBySms`、`forgotPassword` |

### 1.3 行为说明

- 验证码为 **6 位数字**。
- 同一手机号各场景 **60 秒内**不可重复发送。
- 当前仍 **`log.info` 打印验证码**便于本地联调；生产请删除或降级日志，并调用短信 SDK。

## 二、对接真实短信的推荐改法

原则：**HTTP 与业务入口尽量不动**，在 `RedisSmsCodeServiceImpl` 的 `issueLoginSms` / `issueForgotSms` 中于写入 Redis **之后**调用厂商「发送短信」接口即可（验证码仍建议自生成并存 Redis，便于与渠道解耦）。

### 2.1 配置与 SDK

1. 在 `application.yml`（或环境变量）增加厂商参数（示例）：

   ```yaml
   sms:
     enabled: true
     provider: aliyun
     access-key-id: ${SMS_ACCESS_KEY_ID}
     access-key-secret: ${SMS_ACCESS_KEY_SECRET}
     sign-name: 你的签名
     template-login: SMS_xxx
     template-forgot: SMS_yyy
   ```

2. 引入阿里云或腾讯云官方 Maven 依赖（以官网为准）。

3. 在 `issueLoginSms`、`issueForgotSms` 内：生成 `code` → `SET` Redis（保持现有 TTL）→ **调用 SendSms**，模板变量传入 `code`；失败时删除刚写入的 Redis 键并抛出业务异常。

4. **校验逻辑** `verifyLoginSms` / `verifyForgotSms` 一般无需改**（仍读 Redis 比对）**。

### 2.2 与图形验证码的关系

发送短信前已在 `UserServiceImpl` 中调用 `CaptchaService.verifyAndConsume`，对接短信时**无需调整顺序**。
