# 物流查询与支付接口扩展说明

本文说明当前项目中**已预留**的 HTTP 路径、DTO 结构，以及后续如何接入**真实物流查询**与**真实支付网关**（微信 / 支付宝等）。

**支付并行能力（已实现演示链）**：

- **沙箱**：`GET /api/order/pay/mock-data/{id}` + `POST /api/pay/mock/callback`，签名为 `MD5(orderNo + payTradeNo + pay.mock.secret)`。
- **统一下单（与沙箱并行）**：`GET /api/order/pay/unified-data/{id}` + `POST /api/payment/unified/callback`，签名为 `MD5(orderNo + payTradeNo + pay.unified.secret)`；`pay.unified.enabled` 为 `true` 时，`POST /api/payment/prepay` 会返回非 stub 的 `extra`（含 `orderNo`、`payTradeNo`、`sign`）。关闭统一下单演示可将 `pay.unified.enabled` 设为 `false`。

## 一、物流查询（`/api/logistics`）

### 1.1 已实现的路由（占位逻辑）

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| GET | `/api/logistics/order/{orderId}` | 需要登录（JWT） | 校验订单归属当前用户，返回订单内 `logisticsCompany`、`logisticsNo`，`traces` 为空列表，`stub: true` |
| GET | `/api/logistics/query?companyCode=&trackingNo=` | 需要登录（JWT） | 预留：仅回传入参，轨迹为空；对接三方后在此查轨迹 |

### 1.2 响应结构（`LogisticsTrackResponseDTO`）

- `orderNo`：订单号（按单号查询时可为 `null`）
- `logisticsCompany`：承运商名称或编码（与表字段 `order.logistics_company` 或查询参数对齐）
- `logisticsNo`：运单号（对应 `order.logistics_no`）
- `traces`：`LogisticsTraceItemDTO` 列表，字段含 `time`、`status`、`description`
- `stub`：`true` 表示仍为预留/占位实现

### 1.3 数据库与发货流程

订单表已包含：

- `logistics_company`
- `logistics_no`

管理员发货时写入（见管理端发货接口）。用户侧轨迹需调用快递 100、菜鸟、顺丰开放平台等，将返回解析为 `traces` 列表。

### 1.4 后续接入步骤建议

1. **新增配置**：在 `application.yml` 增加物流服务商的 `appKey`、`customer`、请求 URL 等（勿提交密钥到仓库，可用环境变量）。
2. **实现 Client**：例如 `Kuaidi100Client` 封装 HTTP 调用与错误码处理。
3. **新增 Service**：`LogisticsQueryService` 中实现 `queryByTracking(companyCode, trackingNo)` 与可选 `queryByOrderId(userId, orderId)`（内部先查订单再调三方）。
4. **改造 Controller**：将 `LogisticsController` 中组装空 `traces` 的逻辑改为调用上述 Service；对三方超时、限流做降级（返回部分数据或友好错误）。
5. **缓存**：同一运单号短期缓存轨迹 JSON，减轻 QPS 与费用。
6. **安全**：若开放「仅运单号」查询，建议增加登录校验、频率限制或与订单绑定校验，避免被扫号。

## 二、支付网关预留（`/api/payment`）

### 2.1 与现有模拟支付的关系

| 模块 | 路径前缀 | 用途 |
|------|-----------|------|
| 沙箱回调 | `POST /api/pay/mock/callback` | 已实现：验签后更新订单为已支付（开发联调用） |
| 订单内支付 | `PUT /api/order/pay/{id}` 等 | 已实现：本地标记支付等 |
| 网关预留 | `/api/payment/**` | **预留**：统一下单、查单、异步通知占位，对接微信/支付宝时使用 |

### 2.2 已实现的路由

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/api/payment/prepay` | 需要登录 | Body：`PaymentPrepayRequestDTO`（`orderId`、`channel`），返回 `PaymentPrepayResponseDTO`，`stub: true`，业务字段多为 `null` |
| GET | `/api/payment/status?orderId=` | 需要登录 | 返回 `PaymentStatusResponseDTO`；当前 `tradeState` 由本库 `payTime` 推导，**非**渠道真实状态，`stub: true` |
| POST | `/api/payment/notify/alipay` | **无需登录**（已加入 JWT 排除列表） | 占位：固定返回字符串 `success`（支付宝成功应答习惯） |
| POST | `/api/payment/notify/wechat` | **无需登录**（已加入 JWT 排除列表） | 占位：`Content-Type: application/xml`，返回最小成功 XML；接入 V3 时需改为 JSON 等与平台一致 |

### 2.3 DTO 约定

- **预支付请求** `PaymentPrepayRequestDTO`：`orderId`（必填）、`channel`（如 `WECHAT_JSAPI`、`ALIPAY_APP`）。
- **预支付响应** `PaymentPrepayResponseDTO`：微信 JSAPI 常见字段占位（`prepayId`、`paySign` 等）；扩展可用 `extra`。
- **查单响应** `PaymentStatusResponseDTO`：`orderNo`、`tradeState`、`thirdPartyTradeNo`（可与 `order.pay_trade_no` 对齐）。

### 2.4 后续接入步骤建议（微信 / 支付宝）

1. **配置**：商户号、`appId`、API v3 密钥 / 证书路径、支付宝 `appId` 与私钥等，建议使用环境变量或密钥管理服务。
2. **统一下单**：在 `PaymentGatewayController.prepay` 或抽离的 `PaymentGatewayService` 中：
   - 校验订单状态（待支付）、金额与渠道；
   - 调用微信「JSAPI 下单」或支付宝「app 支付接口」；
   - 将返回的调起参数填入 `PaymentPrepayResponseDTO`，并置 `stub: false`。
3. **异步通知**：
   - 在 `notify/alipay`、`notify/wechat` 中解析请求体、**验签**、**幂等**（按渠道交易号去重）；
   - 与 `OrderService.handleMockPayCallback` 类似：更新 `pay_trade_no`、`pay_time`、`status`；
   - 返回渠道要求的字符串或 XML/JSON（失败时返回渠道规定的失败形态，避免重复通知风暴）。
4. **主动查单**：将 `GET /api/payment/status` 改为调用微信「查询订单」/ 支付宝「交易查询」，与本地订单对账；用于回调丢失或前端轮询补偿。
5. **关闭占位**：生产环境去掉 `stub` 或固定为 `false`，并删除 hint 中的预留文案。

### 2.5 JWT 放行说明

`WebMvcConfig` 已对以下路径排除 JWT（与支付回调一致，由渠道服务器调用）：

- `/api/payment/notify/alipay`
- `/api/payment/notify/wechat`

新增其他渠道回调 URL 时，请同步加入 `excludePathPatterns`。

## 三、前端对接说明

项目内已实现页面与 API 封装（订单详情页）：

| 文件 | 说明 |
|------|------|
| `e-commerce-frontend/src/api/logistics.js` | `getLogisticsByOrder`、`queryLogisticsByNumber` |
| `e-commerce-frontend/src/api/payment.js` | `paymentPrepay`、`getPaymentStatus`（与 `src/api/pay.js` 沙箱回调分离） |
| `e-commerce-frontend/src/views/order/OrderDetail.vue` | 物流区块：刷新轨迹、`el-timeline`；支付网关：渠道选择、预支付占位、支付状态描述项 |

- 物流：根据 `traces` 渲染时间轴；`stub === true` 且无节点时展示空状态说明。
- 支付：正式接入后在预支付返回中填充真实字段，再在详情页调用微信 `WeixinJSBridge` / 支付宝 SDK；可继续用 `GET /api/payment/status` 轮询或补单。
